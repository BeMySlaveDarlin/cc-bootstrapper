# BUG-001: Кастомный ввод режима анализа не парсится [RESOLVED v8.1.1]

## Источник
Element чат, тред от 2026-04-06, пользователь Пригожик

## Описание

При выборе глубины анализа (шаг 3B) пользователь выбрал "Other" в AskUserQuestion и ввёл:
```
standart + забери планы из @...
```

Ожидание: режим `standard`, дополнительная инструкция "забери планы из @..." передаётся в контекст.
Результат: запустился режим `deep`, дополнительная инструкция проигнорирована.

## Локализация

### Фаза 3B — Оркестратор (SKILL.md, строки ~143-156)

Оркестратор получает ответ AskUserQuestion. Options:
- `light`
- `standard (рекомендуется)`
- `deep`

Если пользователь выбирает "Other" и пишет произвольный текст — оркестратор должен:
1. Извлечь ключевое слово режима (fuzzy match: `standart` → `standard`, `стандарт` → `standard`)
2. Извлечь дополнительные инструкции (всё после `+`, `,`, `:` или на новой строке)
3. Передать инструкции в `config.custom_instructions`

### Фаза 3C — step-3-apply.md

Поле `analysis_depth` нормализуется (строки 32-33), но только для preset labels.
Нет обработки произвольного текста от "Other".

## Что починить

### 1. SKILL.md — Фаза 3B, после получения ответов AskUserQuestion

Добавить парсинг Other-ответа для вопроса "Анализ":

```
Если ответ на вопрос "Анализ" — произвольный текст (Other):
  1. Извлечь режим: найти первое слово, fuzzy-match к ["light", "standard", "deep"]
     - "standart", "стандарт", "стандартный", "средний" → "standard"
     - "глубокий", "полный", "full" → "deep"
     - "лёгкий", "легкий", "быстрый", "fast" → "light"
     - Не найдено → default "standard"
  2. Извлечь доп. инструкции: всё после первого разделителя (+, запятая, двоеточие, точка с запятой)
     - Trim whitespace
     - Сохранить в поле "custom_instructions" JSON
  3. Подставить нормализованный режим в analysis_depth
```

### 2. step-3-apply.md — Парсинг ответов

Добавить обработку поля `custom_instructions`:

```json
{
  "config": {
    "analysis_depth": "standard",
    "custom_instructions": "забери планы из @...",
    ...
  }
}
```

### 3. step-7-analyze.md — Использование custom_instructions

При запуске глубокого анализа проверить `config.custom_instructions` и передать в prompt субагента.

## Приоритет
High — пользователь теряет контроль над режимом при кастомном вводе.

## Репродукция
1. Запустить `/bootstrap`
2. На шаге 3B выбрать "Other" в вопросе "Глубина анализа"
3. Ввести: `standart + любая инструкция`
4. Проверить `.bootstrap-cache/state.json` → `config.analysis_depth`
