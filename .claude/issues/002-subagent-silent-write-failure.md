# BUG-002: Субагенты молча падают при записи файлов [RESOLVED v8.1.1]

## Источник
Element чат, тред от 2026-04-06, пользователь Пригожик

## Описание

При запуске bootstrap субагенты шага 8 (per-lang генерация) не смогли записать файлы (php и js агенты). Ошибка произошла молча — без уведомления пользователя. Fallback на последовательный режим сработал, но сожрал почасовой лимит токенов.

Цитата:
> "субагентская генерация молча выпала в ошибку, что не смогла записать файлы (php и js агенты), поэтому переделывался в основном потоке"

## Локализация

### Шаг 8 — step-8-lang.md

Субагенты запускаются параллельно (по одному на язык) через Agent tool. Каждый пишет файлы в `.claude/agents/`, `.claude/skills/`, `.claude/pipelines/`.

Возможные причины:
1. **Race condition на mkdir -p** — два субагента одновременно создают `.claude/agents/`
2. **Permissions** — субагент с mode "auto" может не иметь прав на Write
3. **Путь** — субагент неверно резолвит рабочую директорию
4. **Tool denied** — пользователь отклонил Write для одного субагента, и это не обработалось

### Оркестратор (SKILL.md, строки 257-268)

Шаг 8 запускает per-lang агентов параллельно. Если субагент вернул error — оркестратор показывает AskUserQuestion с опциями "Повторить/Пропустить/Остановить" (строки 96-101). Но это generic handler — он не отличает partial success от полного провала.

### bootstrap-team-flow.md (строка 12)

В team mode: "При ошибке team spawn → fallback на sequential для оставшихся шагов". Но fallback переделывает ВСЮ работу с нуля, а не только failed файлы.

## Что починить

### 1. step-8-lang.md — Обработка ошибок записи

Каждая операция Write должна проверяться:

```
Для каждого файла:
  1. Write файл
  2. Проверить результат Write
  3. Если ошибка — добавить в errors[] с путём и причиной
  4. Продолжить со следующим файлом (НЕ останавливаться)
  
В отчёте gen-report-8-{lang}.json — явно указать:
  - "written": [...успешно записанные],
  - "failed": [...неудачные с причиной]
```

### 2. SKILL.md — Шаг 8, обработка partial failure

После получения результатов от параллельных субагентов:

```
1. Прочитать gen-report-8-{lang}.json для каждого языка
2. Собрать все failed файлы
3. Если есть failed:
   - Показать пользователю список failed файлов
   - Предложить: "Повторить failed / Пропустить / Ручной режим"
   - "Повторить" → запустить ТОЛЬКО для failed файлов (не всё заново)
4. Обновить state.json с partial completion
```

### 3. bootstrap-team-flow.md — Granular fallback

Вместо fallback всего шага:
```
При ошибке team agent:
  1. Проверить какие файлы УЖЕ записаны (по gen-report)
  2. Sequential fallback ТОЛЬКО для оставшихся файлов
  3. НЕ переделывать уже записанные
```

### 4. Логирование ошибок

В step-8-lang-log.md обязательно записывать:
- Какие файлы пытались записать
- Какие записались
- Какие упали с ошибкой и ПОЧЕМУ (текст ошибки)

## Приоритет
High — пользователь теряет токены на повторную генерацию без понимания что пошло не так.

## Репродукция
1. Запустить `/bootstrap` на проекте с 2+ языками
2. Дойти до шага 8
3. Наблюдать параллельную генерацию
4. Проверить `.bootstrap-cache/gen-report-8-*.json` на наличие errors
