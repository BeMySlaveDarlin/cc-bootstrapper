# Task() — формат вызова субагента

## Синтаксис

```
Task(<agent-path>, subagent_type: "<type>"):
  Вход: <что читать — файлы, контекст, данные предыдущей фазы>
  Выход: <куда писать результат, ОБЯЗАТЕЛЬНО если генерируется файл>
  Ограничение: <read-only | project-write>
  Верни: summary (<что включить в краткий отчёт>)
```

## Поля

| Поле | Обязательное | Описание |
|------|-------------|----------|
| `agent-path` | да | Путь к агенту: `.claude/agents/{name}.md` |
| `subagent_type` | да | Всегда `"general-purpose"` |
| `Вход` | да | Список входных данных (файлы, контекст) |
| `Выход` | да | Путь к выходному файлу. Если агент пишет в project files — указать "файлы кода" |
| `Ограничение` | да | `read-only` — агент НЕ создаёт/изменяет файлы проекта (src/, app/). `project-write` — агент пишет код в проект |
| `Верни` | да | Формат summary: 5-10 строк, ключевые результаты |

## Правила

- **Write first:** агент ОБЯЗАН записать результат в файл через Write tool ПЕРЕД возвратом summary
- **Summary:** возвращается родительскому pipeline для принятия решений
- **Crash safety:** если агент упал — summary не вернулся, pipeline знает что фаза не завершена
